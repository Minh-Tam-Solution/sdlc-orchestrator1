## Support Request Best Practices

### Before Requesting Support

**✅ Do This First**:
1. **Search Documentation**: Check FAQ, Troubleshooting guide
2. **Try Self-Service**: Follow documented solutions
3. **Ask Team**: Quick questions to colleagues
4. **Gather Information**: Screenshots, error messages, steps to reproduce
5. **Check Status**: Is there a known issue? (Check announcements)

### Writing Good Support Requests

**Good Example**:
```
Subject: [SDLC] Bulk Delete Users Failing with 422 Error

Description:
When attempting to bulk delete users in Admin Panel,
receiving 422 Unprocessable Entity error.

Environment:
- Browser: Chrome 120.0.6099.129
- OS: Windows 11
- User Role: Admin
- Time: 2025-12-20 14:30 UTC

Steps to Reproduce:
1. Navigate to Admin Panel → Users
2. Select 2+ users with checkboxes
3. Click "Delete Selected"
4. Click "Confirm"
5. Receive 422 error

Expected: Users deleted successfully
Actual: 422 error displayed

Console Error:
DELETE /api/v1/admin/users/bulk 422
{"detail": "Unprocessable Entity"}

Screenshots: [attached]

Already Tried:
- Clearing browser cache
- Different browser (Firefox - same issue)
- Single delete (works fine)
- Checked user IDs are valid UUIDs

Urgency: Medium (have workaround - single delete)
Impact: Admin team efficiency (bulk needed for cleanup)
```

**Bad Example**:
```
Subject: It's broken

Description: 
Bulk delete doesn't work. Please fix.
```

### Following Up

**Response Time Guidelines**:
- **Self-Service**: Immediate
- **Team Support**: Same day
- **Admin Support**: Per SLA (1-4 hours depending on priority)
- **Engineering**: 24-48 hours
- **Emergency**: 30 minutes

**When to Follow Up**:
- After expected response time + 25%
- If situation changes (becomes more/less urgent)
- If you discover new information
- If you resolved it yourself (so others know)

**How to Follow Up**:
```
Subject: RE: [Original Subject]

Update:
- [What's changed since original report]
- [Any additional context]
- [Current status]

Still Experiencing:
- [Yes/No]
- [Details if yes]
```

## Tips for Getting Great Support

### 1. Be Specific
- ✅ "Bulk delete users returns 422 error"
- ❌ "Something is broken"

### 2. Provide Context
- ✅ "Using Chrome 120, Admin role, happened at 2PM"
- ❌ "It doesn't work"

### 3. Show What You Tried
- ✅ "Cleared cache, tried Firefox, same issue"
- ❌ "Please fix"

### 4. Be Patient
- ✅ Wait for SLA response time
- ❌ Send multiple messages immediately

### 5. Be Grateful
- ✅ "Thank you for the quick fix!"
- ❌ [Silence after resolution]

### 6. Pay It Forward
- ✅ Help others with same issue
- ✅ Document your solution
- ✅ Update docs if found gap
