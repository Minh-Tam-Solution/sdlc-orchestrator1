## For Developers

### Task 1: Create a New Project

**Time**: 5 minutes  
**Prerequisites**: Developer role or higher

**Steps**:

1. **Navigate to Projects**
   - Click "Projects" in main navigation
   - Or use keyboard shortcut: `P`

2. **Click "Create Project"**
   - Button in top-right corner
   - Or press `N` (New) shortcut

3. **Fill Project Details**
   ```
   Required Fields:
   ├─ Name: "Customer Self-Service Portal"
   ├─ Description: "Enable customers to manage their accounts"
   ├─ Classification: Select tier (LITE/STANDARD/PREMIUM/ENTERPRISE)
   └─ Start Date: 2025-12-20
   
   Optional Fields:
   ├─ Repository URL: https://github.com/org/project
   ├─ Project Lead: Select from dropdown
   ├─ Team Members: Add team
   └─ Tags: customer-portal, self-service
   ```

4. **Choose Classification**
   - **LITE**: Simple tool, 1-2 weeks
   - **STANDARD**: Regular project, 1-3 months ← Most common
   - **PREMIUM**: Large system, 3-6 months
   - **ENTERPRISE**: Mission-critical, 6+ months

5. **Submit**
   - Click "Create Project"
   - Project created with G0 (Foundation) stage active
   - You're redirected to project dashboard

**Result**: Project created, ready for G0 evidence submission

**Next Steps**: [Submit Evidence for G0](#task-2-submit-evidence-for-a-gate)

### Task 2: Submit Evidence for a Gate

**Time**: 10-15 minutes  
**Prerequisites**: Project created, evidence files ready

**Steps**:

1. **Navigate to Project**
   - Go to Projects → Select your project
   - Click on current gate (e.g., "G0: Foundation")

2. **Review Requirements**
   - See "Required Evidence" section
   - Check what artifacts are needed
   - Review AI Council suggestions

3. **Prepare Evidence**
   ```
   For G0 (Foundation):
   ├─ Business case document (PDF/DOCX)
   ├─ Stakeholder analysis spreadsheet (XLSX)
   ├─ Risk assessment document
   └─ Strategic alignment justification
   ```

4. **Upload Evidence**
   - Click "Submit Evidence" button
   - Choose upload method:
     - **File Upload**: Select file from computer
     - **External Link**: Paste URL (GitHub, Google Docs, etc.)
   
5. **Fill Evidence Details**
   ```
   ├─ Title: "G0 Business Case - Customer Portal"
   ├─ Description: Clear explanation of artifact
   ├─ Type: Document/Artifact/Link
   ├─ Gate: G0 (auto-filled)
   └─ Tags: business-case, strategic
   ```

6. **Submit and Verify**
   - Click "Submit"
   - Wait for AI Council review (30-60 seconds)
   - Check feedback/recommendations

7. **Repeat for All Required Evidence**
   - Submit each required artifact
   - Mark as complete when all submitted

**Result**: Evidence stored, AI Council reviewed, visible to approvers

**Next Steps**: [Request Gate Approval](#task-3-request-gate-approval)

---

### Task 3: Request Gate Approval

**Time**: 2 minutes  
**Prerequisites**: All required evidence submitted

**Steps**:

1. **Verify Evidence Complete**
   - Go to gate detail page
   - Check all required evidence submitted
   - Review AI Council completeness check
   - Green checkmark = ready

2. **Click "Request Approval"**
   - Button appears when evidence complete
   - Confirms you want to proceed

3. **Add Review Notes (Optional)**
   ```
   Example:
   "All G0 evidence submitted including:
   - Approved business case ($2M ROI projected)
   - Stakeholder sign-off from VP Product & CFO
   - Risk assessment (medium risk, mitigated)
   
   Ready for review. Target approval: Dec 22."
   ```

4. **Submit Request**
   - Reviewers automatically notified
   - Email sent to assigned reviewers
   - Notification appears in their dashboard

5. **Track Status**
   - Gate status changes to "IN_REVIEW"
   - Check activity feed for updates
   - Receive notification when reviewed

**Result**: Gate in review, reviewers notified

**Timeline**: Typically 1-3 business days for approval

**Next Steps**: Wait for review, [address feedback](#task-4-address-gate-review-feedback) if needed

### Task 4: Address Gate Review Feedback

**Time**: Varies (hours to days)  
**Prerequisites**: Gate review completed with feedback

**Steps**:

1. **Receive Notification**
   - Email notification sent
   - Platform notification appears
   - Gate status shows "NEEDS REVISION"

2. **Read Reviewer Feedback**
   ```
   Navigate to:
   Project → Gate → Reviews Tab
   
   See:
   ├─ Reviewer name
   ├─ Review date
   ├─ Decision (Approved/Rejected/Needs Revision)
   ├─ Feedback comments
   └─ Specific issues identified
   ```

3. **Analyze Feedback**
   ```
   Common Issues:
   ├─ Missing artifacts
   ├─ Incomplete documentation
   ├─ Quality standards not met
   ├─ Compliance gaps
   └─ Clarification needed
   ```

4. **Create Action Plan**
   ```
   For each issue:
   1. Understand the concern
   2. Determine fix approach
   3. Estimate time needed
   4. Prioritize by severity
   ```

5. **Fix Issues**
   - Update existing evidence
   - Upload additional artifacts
   - Provide clarifications
   - Address all feedback points

6. **Document Changes**
   ```
   Add comment explaining fixes:
   "Addressed feedback:
   ✅ Added missing stakeholder approvals
   ✅ Updated risk assessment with mitigation plan
   ✅ Clarified budget justification
   
   Ready for re-review."
   ```

7. **Resubmit**
   - Click "Resubmit for Review"
   - Reviewer notified of updates
   - Review cycle restarts

**Result**: Updated evidence submitted, awaiting re-review

**Tips**: 
- Respond promptly (within 1-2 days)
- Ask questions if feedback unclear
- Over-communicate changes made

**Prerequisites**: Member of project

   **Team Panel**
   ```
   ├─ Project Lead: John Doe
   ├─ Developers: 5 members
   ├─ Reviewers: 2 assigned
   └─ Stakeholders: 3 watching
   ```

   **Recent Activity**
   ```
   ├─ 2h ago: Sarah submitted G2 evidence
   ├─ 1d ago: Mike approved G1
   ├─ 2d ago: AI Council flagged risk
   ```

## For Reviewers

### Task 6: Review Gate Evidence

**Time**: 30-60 minutes per gate  
**Prerequisites**: Reviewer or Admin role

**Steps**:

1. **Access Review Queue**
   - Dashboard → "Pending Reviews" section
   - Or navigate to specific project

2. **Open Gate Review**
   ```
   Click on gate needing review:
   ├─ Project: Customer Portal
   ├─ Gate: G0 Foundation Ready
   ├─ Submitted: 2025-12-18
   └─ Submitted by: John Doe
   ```

3. **Review Evidence**
   ```
   For each artifact:
   ├─ Open/download document
   ├─ Verify completeness
   ├─ Check quality standards
   ├─ Validate compliance
   └─ Cross-reference requirements
   ```

4. **Check AI Council Feedback**
   - Read AI Council assessment
   - Review risk flags
   - Consider recommendations
   - Use as guidance, not decision

5. **Use Review Checklist**
   ```
   G0 Foundation Checklist:
   ☐ Business case clear and justified?
   ☐ Stakeholders identified and aligned?
   ☐ Risks identified and assessed?
   ☐ Strategic alignment demonstrated?
   ☐ Budget/resources justified?
   ☐ Success criteria defined?
   ☐ Compliance requirements identified?
   ```

6. **Make Decision**
   - **Approve**: All criteria met
   - **Needs Revision**: Issues require fixing
   - **Reject**: Fundamental problems

7. **Write Feedback**
   ```
   Good Feedback:
   "Business case well-documented with clear ROI.
   
   Issues to address:
   1. Stakeholder approval from CFO missing (required)
   2. Risk assessment needs quantification (priority: HIGH)
   3. Timeline seems optimistic - add contingency
   
   Decision: NEEDS REVISION
   Expected fix time: 2-3 days
   Happy to re-review once addressed."
   ```

8. **Submit Review**
   - Click "Submit Review"
   - Submitter automatically notified
   - Gate status updated

**Result**: Gate reviewed, feedback provided, submitter notified

**Tips**:
- Be specific and constructive
- Distinguish between must-fix and nice-to-have
- Provide examples or templates
- Set expectations for re-review

### Task 7: Approve/Reject a Gate

**Time**: 5 minutes  
**Prerequisites**: Evidence reviewed, decision made

**Steps**:

1. **Final Verification**
   ```
   ✓ All evidence reviewed
   ✓ Checklist complete
   ✓ AI Council consulted
   ✓ Standards met (or waived)
   ✓ Decision clear
   ```

2. **Select Decision**
   - **Approve**: Pass gate, proceed to next stage
   - **Reject**: Do not pass, requires major rework
   - **Needs Revision**: Minor fixes needed

3. **Write Decision Summary**
   ```
   For Approval:
   "✅ APPROVED
   
   All G0 criteria met:
   - Business case strong ($2M ROI, 827:1 potential)
   - Stakeholder alignment confirmed
   - Risks appropriately assessed
   - Strategic fit validated
   
   Proceed to Stage 01: Planning.
   Next gate: G1 Design Ready"
   ```

4. **Submit Decision**
   - Click "Approve Gate" or "Reject Gate"
   - Confirm decision
   - System updates gate status

5. **Automatic Actions**
   ```
   On Approval:
   ├─ Gate marked PASSED ✓
   ├─ Next stage activated
   ├─ Team notified
   ├─ Audit trail updated
   └─ Compliance records created
   ```

**Result**: Gate decision recorded, project proceeds (or requires fixes)

## For Admins

### Task 8: Create a New User

**Time**: 3 minutes  
**Prerequisites**: Admin role

**Steps**:

1. **Navigate to Admin Panel**
   - Click "Admin" in main navigation
   - Or go to `/admin`

2. **Go to User Management**
   - Admin Panel → Users
   - Click "Create User" button

3. **Fill User Details**
   ```
   Required:
   ├─ Username: john.doe
   ├─ Email: john.doe@company.com
   ├─ Full Name: John Doe
   ├─ Temporary Password: (auto-generated or custom)
   └─ Role: Developer/Reviewer/Admin
   
   Optional:
   ├─ Department: Engineering
   ├─ Team: Backend
   └─ Phone: +84 123 456 789
   ```

4. **Assign Role**
   ```
   Choose one:
   - Developer: Basic access
   - Reviewer: Can approve gates
   - Admin: Full system access
   - Executive: Read-only dashboards
   ```

5. **Set Initial Access**
   ```
   Options:
   ☐ Send invitation email
   ☐ Require password change on first login
   ☐ Enable MFA
   ☐ Add to default projects
   ```

6. **Create User**
   - Click "Create User"
   - User account created
   - Invitation email sent (if enabled)

7. **Verify Creation**
   - Check user appears in user list
   - Verify email sent
   - Test login (optional)

**Result**: New user created and can log in

**Next Steps**: [Add user to projects](#task-10-add-user-to-project)

### Task 9: Bulk Delete Users

**Time**: 2 minutes  
**Prerequisites**: Admin role, production fix deployed (Dec 20, 2025)

**Steps**:

1. **Navigate to User Management**
   - Admin Panel → Users
   - View user list

2. **Select Users to Delete**
   ```
   Options:
   - Click checkboxes for individual users
   - Use "Select All" for all visible
   - Use filters to narrow list first
   ```

3. **Click "Delete Selected"**
   - Button in toolbar
   - Confirmation dialog appears

4. **Confirm Deletion**
   ```
   Warning message shows:
   "You are about to delete X users:
   - john.doe@company.com
   - jane.smith@company.com
   
   This action cannot be undone.
   Users will be permanently removed from all projects."
   ```

5. **Enter Confirmation**
   - Type "DELETE" (case-sensitive)
   - Click "Confirm Delete"

6. **Verify Result**
   ```
   Success message:
   "✅ Successfully deleted 2 users
   - john.doe@company.com (ID: 123)
   - jane.smith@company.com (ID: 456)"
   ```

**Result**: Users permanently deleted from system

**⚠️ Important**:
- **Cannot be undone** - permanent deletion
- Users removed from all projects
- Evidence/artifacts remain (audit trail)
- Use deactivation instead if temporary

**Troubleshooting**:
- If 422 error: See [Troubleshooting Guide](06-Troubleshooting.md#bulk-delete-not-working-fixed)
- Clear cache and hard refresh
- Verify running latest backend version

### Task 10: Add User to Project

**Time**: 1 minute  
**Prerequisites**: Admin or Project Lead role

**Steps**:

1. **Navigate to Project**
   - Projects → Select project
   - Go to "Team" tab

2. **Click "Add Member"**
   - Button in team section
   - User search dialog appears

3. **Search for User**
   ```
   Search by:
   - Name: "John Doe"
   - Email: "john.doe@company.com"
   - Username: "john.doe"
   ```

4. **Select User**
   - Click on user from results
   - User added to selection

5. **Assign Project Role**
   ```
   Options:
   - Member: View and contribute
   - Lead: Manage project
   - Stakeholder: View only
   ```

6. **Add to Project**
   - Click "Add Member"
   - User added to project
   - User notified via email

**Result**: User now has access to project

### Task 11: Generate Compliance Report

**Time**: 5 minutes  
**Prerequisites**: Reviewer or Admin role

**Steps**:

1. **Navigate to Reports**
   - Dashboard → Reports
   - Or Project → Reports tab

2. **Select Report Type**
   ```
   Available Reports:
   ├─ Project Status Report
   ├─ Compliance Report ← Select this
   ├─ Team Performance Report
   └─ Executive Dashboard
   ```

3. **Configure Report Parameters**
   ```
   Filters:
   ├─ Date Range: Last quarter
   ├─ Projects: All or specific
   ├─ Gates: All or specific
   ├─ Status: All/Passed/Failed
   └─ Include: Evidence, Risks, Recommendations
   ```

4. **Preview Report**
   - Click "Preview"
   - Review report content
   - Adjust filters if needed

5. **Choose Export Format**
   ```
   Options:
   - PDF: Professional reports
   - Excel: Data analysis
   - CSV: Raw data
   - JSON: API integration
   ```

6. **Generate Report**
   - Click "Generate Report"
   - Wait for processing (may take 30-60 seconds)
   - Download automatically starts

7. **Review Generated Report**
   ```
   Report contains:
   ├─ Executive Summary
   ├─ Gate Status Overview
   ├─ Evidence Completeness
   ├─ Compliance Score
   ├─ Risk Assessment
   ├─ Audit Trail Summary
   └─ Recommendations
   ```

**Result**: Compliance report generated and downloaded

**Uses**:
- Audits and reviews
- Stakeholder updates
- Compliance verification
- Historical records

## Quick Reference

### Common Workflows

**Starting a Project**
```
1. Create Project (Task 1)
2. Submit G0 Evidence (Task 2)
3. Request Approval (Task 3)
4. Wait for review
5. Address feedback (Task 4)
6. Get approval
7. Move to next stage
```

**Reviewing a Gate**
```
1. Access review queue (Task 6)
2. Review evidence
3. Check AI Council feedback
4. Make decision (Task 7)
5. Provide feedback
6. Submit review
```

**User Management**
```
1. Create user (Task 8)
2. Add to projects (Task 10)
3. Monitor activity
4. Manage roles
5. Deactivate/delete as needed (Task 9)
```
