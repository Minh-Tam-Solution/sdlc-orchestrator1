## 2. Selection Logic (Priority Order)

1. If regulatory/audit pressure >= Moderate → prefer 4.4 (predictive evidence) or at least 4.2+.
2. If AI agents participate beyond coding (planning, quality, ops) → skip directly to 4.4.
3. If team >15 humans or >5 active AI roles → 4.4 for adaptive coordination.
4. If frequent cross-product or multi-tenant scaling → 4.4 (drift + continuity tooling future integration).
5. If only prototyping with <3 people and low risk → 1.x or 2.x snapshot acceptable (timebox to 2 weeks) but plan upgrade.
6. Avoid adopting 4.3 newly; move to 4.4 (4.3 is fully subsumed without losing principles).

## 6. Adoption Decision Flow (Narrative)

1. Assess regulatory pressure & audit traceability requirements.
2. Evaluate AI agent functional scope (coding only vs multi-role predictive support).
3. Determine human + AI team scale and projected growth in 3 months.
4. Select minimal framework version that satisfies evidence + governance without rework.
5. If step 4 output != 4.4, add explicit upgrade checkpoint (date + criteria) in project charter.
