## 🔗 Document Naming Standards Compliance

All SDLC 4.9 documents now comply with:
- **Permanent Naming**: No version numbers in filenames (version inside document)
- **Feature-Based**: Describe WHAT, not WHEN/WHO
- **Kebab-Case**: Consistent formatting
- **AI-Parseable**: Clear, discoverable structure

- ❌ Old: `SDLC-4.8-Executive-Summary.md` (version in filename)

## ⚠️ Important Notes

### Do NOT Use Archived Documents

1. **Reference Only**: These documents are for historical reference only
2. **Use Active Docs**: Always refer to current documents in `/01-Overview`, `/02-Core-Methodology`, etc.
3. **Version Information**: Version history is maintained in Git, not in filenames
4. **Breaking Changes**: SDLC 4.9 is backward compatible but significantly enhanced

### Migration Process

When migrating documents from SDLC 4.8 → 4.9:

1. **Archive First**: Copy old document to `99-Legacy/SDLC-4.8-Archive/`
2. **Update Content**: Enhance with 10-stage methodology where applicable
3. **Rename File**: Remove version from filename (e.g., `SDLC-4.8-X` → `SDLC-X`)
4. **Update Version**: Change version inside document to 4.9.0
5. **Update Links**: Ensure all internal links use new permanent names
6. **Delete Old**: Remove old document from active directory
