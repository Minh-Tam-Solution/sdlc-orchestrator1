## 🔍 What Goes Here

### Should be in 99-Legacy:
- Previous SDLC versions (1.x through 4.6)
- Superseded governance specifications
- Old validation tools
- Deprecated templates
- Design-first artifacts (replaced by battle-tested)
- Emergency patches that are now integrated

### Should NOT be in 99-Legacy:
- Active legacy management tools (stay in scripts/legacy-governance/)
- Reference documentation still useful
- Battle-tested patterns (these are current)
- Crisis response tools (still active)

## 📋 Retention Policy

### Permanent Retention:
- Version evolution history
- Crisis resolution documentation
- Major architectural decisions
- Lessons learned documents

### Review for Removal (After 1 Year):
- Duplicate content
- Failed proposals
- Temporary patches
- Draft documents

## 🔧 Management Tools

Use the scripts in `/scripts/legacy-governance/` to:
- Scan for legacy content: `legacy_scan.py`
- Check integrity: `legacy_hash_update.py`
- Monitor drift: `legacy_drift_check.py`
- Review retention: `legacy_retention_review.py`
