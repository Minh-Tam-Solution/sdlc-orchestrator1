### **4. System Thinking Integration (Pillar 3 Core)**

**Iceberg Model - 4 Layers**:
```
Layer 1: EVENTS (visible symptoms) ← Most developers stop here
Layer 2: PATTERNS (recurring themes) ← Traditional analysis
Layer 3: STRUCTURES (systems/processes) ← Design solutions here
Layer 4: MENTAL MODELS (beliefs/assumptions) ← Design Thinking empathy reaches here
```

**Critical Connection** (MUST preserve in 4.9):
- Design Thinking EMPATHIZE → Layer 4 (Mental Models)
- Design Thinking DEFINE → Layer 3 (Structures)
- Traditional SDLC → Layer 1-2 (Events, Patterns)

**Example** (NQH-Bot - preserve this):
```
Layer 1: "Attendance tracking is slow" (event)
Layer 2: "Managers spend 2 hours daily" (pattern)
Layer 3: "Manual process creates 5-10% errors" (structure)
Layer 4: "I don't trust the data anymore" (mental model) ← DT finds this!

Solution: Build trust through verifiable proof (GPS + photo)
Result: 90% trust restoration (root problem solved)
```

### **5. Design Thinking Quality Gates (0.1-0.5)**

**MUST PRESERVE in 4.9** (these are battle-tested):

```yaml
Gate 0.1: Problem Definition Validated

Gate 0.2: Solution Diversity Validated
  - 100+ ideas generated
  - 10+ distinct themes
  - Wild ideas included (challenge assumptions)

Gate 0.3: Prototype Fidelity
  - Minimum to LEARN (not minimum to SHIP)
  - Started at lowest fidelity
  - Real users can test

Gate 0.4: Test Validity
  - Tested with real users (not proxies)
  - 5-8 participants minimum
  - Real context (not artificial lab)

Gate 0.5: Ship Decision
  - SHIP: >80% task completion, assumptions validated
  - ITERATE: 50-80% completion, fixable issues
  - PIVOT: <50% completion, fundamental issues
```
