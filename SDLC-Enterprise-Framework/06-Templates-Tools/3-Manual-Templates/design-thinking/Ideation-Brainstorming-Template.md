# 💡 Ideation & Brainstorming Template

**Purpose**: Generate diverse solution ideas through structured creativity techniques
**When to Use**: IDEATE phase (SDLC Stage 02 - HOW?)
**Time Required**: 60-90 minutes
**Participants**: 5-8 people (product, design, eng, business, customer-facing roles)

---

## 📋 How to Use This Template

1. **Complete HMW Questions first** - Use top 3-5 HMWs as prompts
2. **Defer judgment** - No criticism during generation
3. **Go for quantity** - 100+ ideas minimum
4. **Build on others' ideas** - "Yes, and..." instead of "No, but..."
5. **Encourage wild ideas** - Crazy ideas can lead to breakthrough solutions

## 🎯 SESSION AGENDA (90 minutes)

### Part 1: Warm-Up (10 min)

**Goal**: Get creative juices flowing, build psychological safety

**Activity: Crazy 8s Warm-Up**
```
1. Give everyone blank paper folded into 8 sections
2. Prompt: "Draw 8 ways to use a paperclip" (silly, non-work)
3. Time: 1 minute per idea (8 min total)
4. Share: Everyone shows their 8 ideas (2 min)
```

**Why This Works**: Loosens up brain, demonstrates quantity over quality, builds trust

---

### Part 2: Ideation Round 1 (20 min)

**HMW Prompt**: [Top HMW #1 - highest impact]

**Example (NQH-Bot)**:
```
HMW: How might we capture GPS + photo + timestamp automatically
     to eliminate 45-minute verification?
```

**Technique: Silent Brainstorming → Share**

**Minutes 1-10: Silent Generation**
```
- Everyone silently writes ideas on sticky notes
- One idea per sticky note
- 3-5 words per idea (headline format)
- Target: 10 ideas per person in 10 minutes
```

**Minutes 11-15: Share & Cluster**
```
- Each person reads their ideas aloud (30 sec per person)
- Stick notes on wall
- Facilitator groups similar ideas into clusters
```

**Minutes 16-20: Build & Combine**
```
- Identify most interesting clusters
- "Yes, and..." - build on promising ideas
- Combine ideas to create hybrids
- Target: 5-10 hybrid concepts
```

**Output**: 50-80 ideas for HMW #1

---

### Part 3: Ideation Round 2 (20 min)

**HMW Prompt**: [Top HMW #2]

**Example (NQH-Bot)**:
```
HMW: How might we reduce manager admin time from 2 hours to <10 minutes?
```

**Technique: SCAMPER Method**

**SCAMPER Framework**:
```
S = Substitute - What can we replace? (5 min)
  Example: Substitute manager verification with AI verification

C = Combine - What can we merge? (3 min)
  Example: Combine check-in with shift schedule confirmation

A = Adapt - What can we adjust/copy from elsewhere? (3 min)
  Example: Adapt Uber's real-time tracking to staff presence

M = Modify - What can we change (size, shape, meaning)? (3 min)
  Example: Modify weekly timesheet to instant micro-confirmations

P = Put to other uses - New ways to use existing things? (2 min)
  Example: Use staff smartphones (already have) as check-in devices

E = Eliminate - What can we remove? (2 min)
  Example: Eliminate paper forms, manual entry, verification step

R = Reverse/Rearrange - What if we did opposite/different order? (2 min)
  Example: Reverse flow: Staff verify managers (funny but sparks ideas!)
```

**Output**: 50-80 ideas for HMW #2

---

### Part 4: Ideation Round 3 (20 min)

**HMW Prompt**: [Top HMW #3]

**Example (NQH-Bot)**:
```
HMW: How might we make verification transparent (not surveillance)?
```

**Technique: Analogies & Cross-Pollination**

**Minutes 1-5: Identify Analogous Domains**
```
What other industries solve similar problems?
- Banking: Identity verification (biometric, 2FA)
- Uber: Driver/passenger trust (ratings, real-time tracking)
- Social Media: Content authenticity (blue checkmarks)
- Gaming: Achievement verification (blockchain NFTs)
- Healthcare: Patient identity (wristbands, biometric)
```

**Minutes 6-15: Borrow Solutions**
```
For each domain, ask: "How might we solve our problem like [industry] does?"

Banking → Biometric Verification:
- Face ID check-in
- Fingerprint timestamp
- Two-factor attendance (manager approval + GPS)

Uber → Real-Time Trust:
- Staff can see their own location data
- Transparent check-in map (everyone sees everyone)
- Rating system (staff rate check-in UX)

Social Media → Authenticity Badges:
- "Verified Check-In" badge for accurate submissions
- Transparency dashboard (staff see verification criteria)
- Gamification (streak badges for consistent accuracy)
```

**Minutes 16-20: Adapt & Refine**
```
Pick 3-5 most promising borrowed solutions
Adapt to our specific context (F&B, Vietnamese culture, manager-staff dynamics)
```

**Output**: 30-50 ideas for HMW #3

---

### Part 5: Cluster & Prioritize (15 min)

**Goal**: Organize 130-210 ideas into actionable concepts

**Step 1: Affinity Mapping (5 min)**
```
Group similar ideas into themes
Target: 10-15 themes
```

**Example Themes (NQH-Bot)**:
```
Theme 1: Biometric Verification (12 ideas)
Theme 2: GPS Auto-Capture (18 ideas)
Theme 3: Visual Proof (photo/video) (15 ideas)
Theme 4: Blockchain Audit Trail (8 ideas)
Theme 5: Real-Time Dashboards (10 ideas)
Theme 6: Transparent Verification (staff visibility) (14 ideas)
Theme 7: Gamification (7 ideas)
Theme 8: AI Anomaly Detection (9 ideas)
Theme 9: Peer Verification (6 ideas)
Theme 10: Manager Alerts (11 ideas)
```

**Step 2: Dot Voting (5 min)**
```
Each person gets 3 votes (dot stickers or tally marks)
Vote for themes that are:
  - Highest impact on user problem
  - Most exciting/innovative
  - Feasible to prototype quickly

Tally votes
```

**Step 3: Select Top 3 Concepts (5 min)**
```
Pick 3 themes with highest votes
These become prototyping candidates
```

**Example Results (NQH-Bot)**:
```
🥇 GPS Auto-Capture + Photo (18 ideas, 8 votes)
🥈 Real-Time Dashboards (10 ideas, 7 votes)
🥉 Transparent Verification (14 ideas, 6 votes)
```

### Part 6: Concept Sketching (10 min)

**Goal**: Visualize top 3 concepts at high level

**For each of top 3 concepts:**
```
1. Name the concept (3-5 words)
2. Draw simple sketch (UI mockup, flow diagram, or storyboard)
3. Write 1-sentence value proposition
4. List 3 key features
```

**Example (NQH-Bot - GPS Auto-Capture)**:

```
Concept Name: TrustCheck - Auto-Verify Attendance

Sketch:
┌─────────────────────────────────┐
│  [Photo of staff member]        │
│  Name: Nguyen Van A             │
│  Location: Restaurant B         │
│  Time: 7:32am                   │
│  GPS: ✅ Verified               │
│  [Confirm Check-In] Button      │
└─────────────────────────────────┘

Value Proposition:
Staff check in via smartphone with automatic GPS + photo verification,
eliminating 45 minutes daily of manager verification time.

Key Features:
1. One-tap check-in with GPS + photo auto-capture
2. Real-time manager dashboard (no 2-hour delay)
3. Immutable audit trail (blockchain-style transparency)
```

## 🎨 IDEATION TECHNIQUES REFERENCE

### Technique 1: Brainstorming (Classic)
**Best for**: Generating high quantity of ideas quickly
**Duration**: 15-20 minutes
**Process**: Silent writing → Share → Build on ideas

---

### Technique 2: Crazy 8s
**Best for**: Rapid sketching, visual thinkers
**Duration**: 8 minutes
**Process**: Fold paper into 8 sections, 1 minute per sketch

**Example Prompt**:
```
"Draw 8 different ways to verify staff attendance"

Ideas might include:
1. Face ID check-in
2. QR code scan at location
3. Bluetooth beacon proximity
4. Photo with GPS geotag
5. Voice confirmation
6. Wearable device (smartwatch)
7. Peer verification (staff confirm each other)
8. AI video analysis (security cameras)
```

---

### Technique 3: SCAMPER
**Best for**: Systematic exploration, improving existing solutions
**Duration**: 15-20 minutes
**Process**: Apply 7 lenses (Substitute, Combine, Adapt, Modify, Put to use, Eliminate, Reverse)

---

### Technique 4: Analogies & Cross-Pollination
**Best for**: Borrowing proven solutions from other domains
**Duration**: 20-30 minutes
**Process**: Identify analogous problems → Borrow solutions → Adapt to context

**Example Industries to Borrow From**:
```
- Banking (security, verification)
- Healthcare (patient safety, compliance)
- Gaming (engagement, rewards)
- Social media (trust, authenticity)
- Transportation (tracking, safety)
- E-commerce (user experience, conversion)
```

---

### Technique 5: Provocation & "What If"
**Best for**: Breaking assumptions, wild ideas
**Duration**: 10-15 minutes
**Process**: Ask provocative "What if" questions

**Example Provocations (NQH-Bot)**:
```
What if... attendance tracking didn't exist?
What if... we paid staff by output instead of hours?
What if... customers verified staff presence (not managers)?
What if... attendance data was public (transparent to everyone)?
What if... check-in process was fun (gamified)?
What if... AI predicted no-shows before they happen?
```

---

### Technique 6: Reverse Thinking
**Best for**: Challenging status quo
**Duration**: 10-15 minutes
**Process**: Flip the problem → Generate "worst possible solutions" → Reverse them

**Example (NQH-Bot)**:
```
Reverse Problem: "How might we make attendance tracking as painful as possible?"

Worst Solutions:
- Require staff to visit office 3x/day for verification
- Make check-in process take 15 minutes
- Randomize check-in process daily (unpredictable)
- Punish accurate check-ins (perverse incentive)

Reverse to Good Solutions:
- Remote check-in (staff never visit office)
- One-tap check-in (<10 seconds)
- Consistent, predictable process
- Reward accurate check-ins (gamification)
```

---

### Technique 7: Brainwriting (Silent Brainstorming)
**Best for**: Introverts, remote teams, avoiding groupthink
**Duration**: 15-20 minutes
**Process**: Silent individual writing → Share → Build

**Why This Works**:
- Prevents loud voices from dominating
- Allows deep thinking (not reactive)
- Generates more diverse ideas
- Works well for distributed teams

## ✅ IDEATION SESSION QUALITY CHECKLIST

### Good Ideation Sessions Have:

- [ ] **High Quantity**: 100+ ideas generated
- [ ] **Diverse Ideas**: Mix of incremental + breakthrough concepts
- [ ] **Equal Participation**: Everyone contributed (not dominated by 1-2 people)
- [ ] **Wild Ideas**: At least 10-20 "crazy" ideas that challenge assumptions
- [ ] **Builds on Others**: Hybrid concepts combining multiple ideas
- [ ] **Visual Elements**: Sketches/diagrams (not just words)
- [ ] **Energy**: Team felt energized, not drained

### Red Flags (Fix for Next Session):

- ❌ **Low Quantity**: <50 ideas (team too critical, not enough time)
- ❌ **Safe Ideas**: All incremental, no wild concepts (fear of judgment)
- ❌ **Dominated**: 1-2 people generated 80% of ideas (facilitator failed)
- ❌ **Criticism**: "That won't work" during generation (broke ground rules)
- ❌ **Drained Energy**: Team felt exhausted, not excited (session too long or negative)

## 💡 PRO TIPS

### Tip 1: Set a Quota
> "Require each person to generate minimum 10 ideas. Quota forces quantity, which unlocks quality."

### Tip 2: Embrace Silence
> "Silent brainstorming first, then share. Prevents groupthink and gives introverts space to think."

### Tip 3: Mix Roles
> "Invite customer support, sales, ops (not just product/eng). Diverse perspectives unlock breakthrough ideas."

### Tip 4: Take Breaks
> "If energy dips, take 5-minute break. Brain needs rest to sustain creativity for 90 minutes."

### Tip 5: Capture Everything
> "Even 'bad' ideas can spark great ones. Write everything down, defer judgment."

### Tip 6: Use Constraints to Spark Creativity
> "What if budget was $0? What if we had to ship in 1 week? Constraints force creative solutions."
