================================================================================
LESSONS LEARNED
================================================================================

1. CLI Parameter Discovery
   - Issue: Used incorrect parameter names (--config-path, --private-key-path)
   - Solution: Always run --help to discover actual CLI parameters
   - Impact: Saved debugging time on future tests

2. CLI Output Format Variability
   - Issue: Expected exact text like "Disconnected from slack"
   - Actual: CLI outputs "Slack disconnected successfully"
   - Solution: Check for key words in lowercase instead of exact match
   - Pattern: assert ("disconnected" in output.lower() and "slack" in output.lower())

3. Evidence Vault Isolation
   - Issue: CLI creates its own Evidence Vault instance
   - Attempted: Verify artifacts in test fixture's vault
   - Lesson: Each CLI invocation has isolated Evidence Vault
   - Solution: Verify via output messages and exit codes instead

4. Command Syntax Discovery
   - Issue: Used --platform flag for disconnect/test commands
   - Actual: Commands use --slack or --github flags directly
   - Impact: Discovered by running commands and reading error messages
   - Benefit: Tests now match real CLI usage patterns

5. Performance Testing Efficiency
   - Result: All 8 tests execute in 0.22s (110x faster than budget)
   - Insight: Mock API responses are extremely fast
   - Benefit: Provides huge headroom for future test additions
   - Note: Real API calls would be slower (need separate performance tests)

================================================================================
INTEGRATION TEST ARCHITECTURE
================================================================================

Test Pattern: E2E Lifecycle Testing

Pattern Components:
  1. Fixture Setup
     - tempfile.TemporaryDirectory() for isolated configs
     - EvidenceVaultAdapter with generated keys
     - Mock API response fixtures

  2. CLI Invocation
     - typer.testing.CliRunner for command execution
     - Real CLI commands (no direct function calls)
     - Output and exit code verification

  3. API Mocking
     - unittest.mock.patch for external API calls
     - Mock Slack API responses (auth.test, conversations.list)
     - Mock GitHub API responses (installation, repositories)

  4. Verification
     - Exit code assertions (0 = success)
     - Output text verification (key words in lowercase)
     - Configuration file validation (.mcp.json)
     - Performance timing measurements

  5. Cleanup
     - Disconnect commands with --force flag
     - Verify platform removed from config
     - Temporary directories auto-deleted

Example Test Structure:
```python
def test_slack_mcp_full_lifecycle(temp_mcp_config, evidence_vault):
    start_time = time.time()

    # Step 1: Connect
    result = runner.invoke(app, ['connect', '--slack', ...])
    assert result.exit_code == 0

    # Step 2: Verify config
    config = json.loads((temp_mcp_config / ".mcp.json").read_text())
    assert "slack" in config["platforms"]

    # Step 3: Test connection
    result = runner.invoke(app, ['test', '--slack', ...])
    assert result.exit_code == 0

    # Step 4: Disconnect
    result = runner.invoke(app, ['disconnect', '--slack', '--force', ...])
    assert result.exit_code == 0

    # Step 5: Verify performance
    elapsed = time.time() - start_time
    assert elapsed < 5.0
```
